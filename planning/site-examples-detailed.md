## Key Findings for EventFinder

### Website Diversity
- **32 individual sites analyzed** (21 music, 11 craft)
- **67% success rate** for music venues (14/21 accessible)
- **73% success rate** for craft workshops (8/11 accessible)
- **Overall: 69% of sites have extractable event/class data**

### Common Extraction Challenges
1. **404 Errors**: 3 sites (outdated links from directories)
2. **No Public Calendar**: 3 venues (rental-only or private)
3. **Technical Issues**: 2 sites (SSL errors, unclear structure)
4. **Booking-Focused**: 2 sites (not public-facing calendars)

### Integration Patterns Found
- Custom HTML listings: Most common (18 sites)
- Third-party API: 2 sites (Showpass, Ticketmaster)
- Google Calendar embed: 1 site
- Shopify collections: 2 sites (craft workshops)
- On-demand booking systems: 2 sites

### Data Quality Insights
- **High consistency**: Event name, date (when calendar exists)
- **Medium consistency**: Prices, times, descriptions
- **Low consistency**: Images, ticket links, detailed requirements

### Recommended Approach
1. **Playwright MCP primary tool**: Can handle all site types
2. **LLM extraction**: Essential for diverse formats and natural language dates
3. **Error handling**: Must gracefully handle 404s and missing data
4. **Flexible schema**: Events vary from 2-hour workshops to 8-week series
5. **URL validation**: Directory links may be outdated
6. **Format detection**: Different handling for on-demand vs scheduled events
